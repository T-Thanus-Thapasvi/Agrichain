# Manager Dashboard - API Integration Details

## Overview
The Manager Dashboard integrates with the AgriChain API Gateway using the existing HTTP interceptor for authentication. All endpoints use relative paths; the base URL and JWT token are handled automatically.

## Service Layer

### SubsidyService (`src/app/services/subsidy.service.ts`)

```typescript
getAllPrograms(): Observable<SubsidyProgram[]>
updateProgram(program: SubsidyProgramUpdateRequest): Observable<SubsidyProgram>
deleteProgram(programID: number): Observable<void>
```

All methods use relative API paths (no hardcoded base URL).

## API Endpoints

### 1. Fetch All Subsidy Programs

**Endpoint:** `GET /subsidy-programs`

**Purpose:** Load all subsidy programs for the grid display

**Request:** None

**Response:**
```json
[
  {
    "programID": 1,
    "title": "Farm Input Subsidy 2025",
    "description": "Subsidy for agricultural inputs and equipment",
    "startDate": "2025-01-15",
    "endDate": "2025-12-31",
    "allottedBudget": 5000000,
    "consumedBudget": 2300000,
    "subsidyStatus": "PENDING"
  },
  ...
]
```

**UI Usage:** 
- Displayed in programs grid
- Used to calculate statistics
- Loaded on component init and after any CRUD operation

---

### 2. Create New Subsidy Program

**Endpoint:** `POST /subsidy-programs`

**Purpose:** Create a new subsidy program

**Request Body:**
```json
{
  "title": "Rural Development Fund",
  "description": "Support for rural area development initiatives",
  "startDate": "2025-02-01",
  "endDate": "2025-11-30",
  "allottedBudget": 3500000,
  "subsidyStatus": "PENDING"
}
```

**Notes:**
- Do NOT include `programID` (auto-generated by backend)
- Do NOT include `consumedBudget` (auto-set by backend)
- All dates in `YYYY-MM-DD` format
- Status must be one of: `PENDING`, `VALIDATED`, `REJECTED`
- Budget must be ≥ 1

**Response:**
```json
{
  "programID": 5,
  "title": "Rural Development Fund",
  "description": "Support for rural area development initiatives",
  "startDate": "2025-02-01",
  "endDate": "2025-11-30",
  "allottedBudget": 3500000,
  "consumedBudget": 0,
  "subsidyStatus": "PENDING"
}
```

**UI Usage:**
- Modal form submission
- On success: Modal closes, toast shows confirmation, grid refreshes
- On error: Toast shows error message, modal stays open for correction

---

### 3. Update Subsidy Program

**Endpoint:** `PUT /subsidy-programs/{id}`

**Purpose:** Update an existing subsidy program

**Request Path Parameter:**
- `id`: Program ID (from URL: `/subsidy-programs/{id}`)

**Request Body:**
```json
{
  "programID": 1,
  "title": "Farm Input Subsidy 2025 - Updated",
  "description": "Subsidy for agricultural inputs and equipment - Revised",
  "startDate": "2025-01-15",
  "endDate": "2025-12-31",
  "allottedBudget": 5500000,
  "subsidyStatus": "PENDING"
}
```

**Notes:**
- Include `programID` in request body (must match URL parameter)
- Do NOT include `consumedBudget` (read-only)
- All fields must be provided (full update required)
- All dates in `YYYY-MM-DD` format

**Response:** Updated program object (same structure as Create response)

**UI Usage:**
- Edit modal form submission
- Pre-populated with existing program values
- On success: Modal closes, toast shows confirmation, grid refreshes
- On error: Toast shows error message, modal stays open for correction

---

### 4. Delete Subsidy Program

**Endpoint:** `DELETE /subsidy-programs/{id}`

**Purpose:** Delete a subsidy program

**Request Path Parameter:**
- `id`: Program ID to delete

**Request Body:** None

**Response:** 204 No Content (empty response)

**Precondition:** 
- User must confirm deletion in modal first
- No validation on backend side that we control

**UI Usage:**
- Delete button in program card
- Shows confirmation dialog first
- On user confirmation: Calls DELETE API
- On success: Toast shows confirmation, grid refreshes
- On error: Toast shows error message

## Authentication & Headers

### Automatic Token Handling
The existing HTTP Interceptor (`src/app/core/interceptors/http-interceptor.ts`) automatically:
- Extracts JWT token from `localStorage` (key: `agrichain_token`)
- Attaches token as: `Authorization: Bearer {token}`
- No manual header configuration needed in SubsidyService

## Error Handling

### HTTP Error Responses

**400 Bad Request**
```json
{
  "error": "Invalid subsidy program data",
  "message": "Budget must be greater than 0"
}
```
**UI Handling:** Toast shows error message

**401 Unauthorized**
- Token expired or invalid
- **UI Handling:** User redirected to login (auth guard)

**403 Forbidden**
- User role doesn't have permission
- **UI Handling:** Access denied (role guard prevents access)

**404 Not Found**
```json
{
  "error": "Program not found",
  "message": "Subsidy program with ID 999 does not exist"
}
```
**UI Handling:** Toast shows error message

**500 Internal Server Error**
```json
{
  "error": "Server error",
  "message": "An unexpected error occurred"
}
```
**UI Handling:** Toast shows generic error message

### Component Error Handling
```typescript
.subscribe({
  next: (data) => { /* Success */ },
  error: (error) => {
    // Attempts to show backend message
    const message = error?.error?.message || 'Default error message';
    this.toast.showError(message);
  }
})
```

## Data Models

### SubsidyProgram (Read Response)
```typescript
interface SubsidyProgram {
  programID: number;           // Auto-generated by backend
  title: string;               // User-entered
  description: string;         // User-entered
  startDate: string;           // YYYY-MM-DD format
  endDate: string;             // YYYY-MM-DD format
  allottedBudget: number;      // In Rupees, user-entered
  consumedBudget: number;      // Auto-calculated by backend
  subsidyStatus: 'PENDING' | 'VALIDATED' | 'REJECTED'; // User-selectable
}
```

```typescript
interface SubsidyProgramCreateRequest {

  startDate: string;
  endDate: string;
  allottedBudget: number;
  subsidyStatus: 'PENDING' | 'VALIDATED' | 'REJECTED';
}
```

### SubsidyProgramUpdateRequest (Write Request)
```typescript
interface SubsidyProgramUpdateRequest extends SubsidyProgramCreateRequest {
  programID: number; // Required for update

```

| `VALIDATED` | Closed – Budget Full | Budget allocation is complete |

## Date Format Specifications

### Storage Format
- **Backend**: YYYY-MM-DD (ISO 8601)
- **Example**: "2025-01-15"

### Display Format (in UI)
- **Format**: DD MMM YYYY
- **Example**: "15 Jan 2025"
- **Implementation**: Uses `Intl.DateTimeFormat` with 'en-IN' locale

### Form Input
- Uses HTML5 date picker
- Browser handles local formatting
- Always sends YYYY-MM-DD to backend

---

## Currency Format Specifications

### Storage Format
- **Backend**: Numeric value in Rupees (no decimal points)
- **Example**: 5000000 (₹50 lakhs)

### Display Format
- **Format**: Indian numbering with ₹ symbol
- **Example**: ₹50,00,000
- **Implementation**: Uses `Intl.NumberFormat` with 'en-IN' locale

## Validation Rules

### Form-Side Validation (Before API Call)
1. **Title**: Non-empty string
2. **Description**: Non-empty string
3. **Budget**: Number ≥ 1
4. **Start Date**: Valid date, required
5. **End Date**: Valid date, required, must be after Start Date
6. **Status**: Must be selected from enum values

### Backend Validation (API Response)
- Additional business logic validation
- Budget constraints
- Date range constraints
- Status transition rules

## Request/Response Timing

### Typical Flow
1. User action (Create/Edit/Delete)
2. Form validation (0ms)
3. API request sent (~50-500ms network time)
4. Backend processing (50-2000ms)
5. Response received
6. Component updates signals
7. UI re-renders via Angular change detection

## Common Integration Issues & Solutions

| Issue | Cause | Solution |
|---|---|---|
| 401 Unauthorized | Missing/expired token | Check localStorage for valid token, re-login |
| 403 Forbidden | User lacks MANAGER role | Verify user role in JWT payload |
| 404 Not Found | Invalid program ID | Refresh grid, verify program ID exists |
| 400 Bad Request | Invalid data format | Check field types and format (dates as YYYY-MM-DD) |
| Network timeout | API unreachable | Verify API Gateway is running, check network |
| CORS error | Cross-origin issue | Should be handled by proxy/API Gateway |
